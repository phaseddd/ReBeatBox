    **F. Add registerKeyboardDispatcher() private method:**

    ```java
    /**
     * Registers a global key event dispatcher via KeyboardFocusManager
     * that intercepts KEY_PRESSED/KEY_RELEASED events and routes them
     * through KeyboardMapper -> RealtimeReceiver -> NoteEventBus -> KeyboardHintPanel.
     *
     * <p>Per D-03: uses boolean[128] dedup via KeyboardMapper.isActive().
     * Returns false to never consume events (per RESEARCH.md anti-pattern warning).
     * Text component focus guard prevents MIDI triggers during text input.
     */
    private void registerKeyboardDispatcher() {
        KeyboardFocusManager kfm = KeyboardFocusManager.getCurrentKeyboardFocusManager();
        kfm.addKeyEventDispatcher(e -> {
            // Step 1: Only KEY_PRESSED and KEY_RELEASED (ignore KEY_TYPED)
            int id = e.getID();
            if (id != KeyEvent.KEY_PRESSED && id != KeyEvent.KEY_RELEASED) {
                return false;
            }

            // Step 2: Respect already-consumed events
            if (e.isConsumed()) {
                return false;
            }

            // Step 3: Text component focus guard (Pitfall 2)
            Component focusOwner = kfm.getFocusOwner();
            if (focusOwner instanceof JTextComponent) {
                return false;
            }

            // Step 4: Look up MIDI note from key mapping
            int note = KeyboardMapper.keyCodeToNote(e.getKeyCode());
            if (note < 0) {
                return false; // unmapped key
            }

            // Step 5: Deduplicate + send MIDI + fire events + visual highlight
            boolean pressed = (id == KeyEvent.KEY_PRESSED);
            if (pressed && !keyboardMapper.isActive(note)) {
                receiver.noteOn(note, 100);                 // Channel 0, velocity 100 (D-02)
                eventBus.fireLiveNoteOn(note, 100);         // D-10
                keyboardMapper.setActive(note, true);       // dedup (D-03)
                keyboardHintPanel.setKeyHighlighted(e.getKeyCode(), true); // D-08
            } else if (!pressed && keyboardMapper.isActive(note)) {
                receiver.noteOff(note);                     // Channel 0
                eventBus.fireLiveNoteOff(note);             // D-10
                keyboardMapper.setActive(note, false);
                keyboardHintPanel.setKeyHighlighted(e.getKeyCode(), false);
            }

            return false; // Never consume — allow other components to process (D-03)
        });
    }
    ```

    **G. Add registerFocusLossHandler() private method:**

    ```java
    /**
     * Registers a window focus listener that cleans up all active live notes
     * and visual highlights when the window loses focus (Alt+Tab, etc.).
     *
     * <p>Prevents "stuck notes" (Pitfall 1): when the user switches windows
     * while holding a key, KEY_RELEASED never fires. This handler sends
     * NoteOff for all active notes and clears the keyboard hint panel.
     */
    private void registerFocusLossHandler() {
        addWindowFocusListener(new WindowAdapter() {
            @Override
            public void windowLostFocus(WindowEvent e) {
                if (keyboardMapper == null || receiver == null) return;

                // Send NoteOff for all active live notes
                for (int note = 0; note < 128; note++) {
                    if (keyboardMapper.isActive(note)) {
                        receiver.noteOff(note);
                        eventBus.fireLiveNoteOff(note);
                        keyboardMapper.setActive(note, false);
                    }
                }

                // Clear visual highlights
                if (keyboardHintPanel != null) {
                    keyboardHintPanel.clearAllHighlights();
                }
            }
        });
    }
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AWT KeyEvent → KeyboardFocusManager dispatcher | Untrusted input enters the system through OS-level keyboard events. AWT validates event structure; MIDI note range validated by KeyboardMapper (-1 sentinel for unmapped). |
| Window focus event → WindowFocusListener | System event from AWT. No external data crosses this boundary. |
| Component tree → SidebarPanel.getContentPanel() | Public accessor exposes internal JPanel. Only called by ReBeatBoxWindow in same package. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-15 | Spoofing | KeyboardFocusManager.addKeyEventDispatcher() | accept | Key events originate from AWT system queue (OS-validated). A single-user desktop app cannot inject fake KeyEvents via external means without process-level access. |
| T-03-16 | Denial of Service | KeyEventDispatcher lambda | mitigate | Text component focus guard (`focusOwner instanceof JTextComponent`) prevents MIDI triggers during text input. `boolean[128]` dedup via KeyboardMapper prevents OS auto-repeat MIDI storm. Key lookup is O(1) HashMap get — dispatch < 1ms. |
| T-03-17 | Denial of Service | WindowFocusListener.windowLostFocus() | mitigate | Cleanup loop iterates 128 entries (boolean array scan) — < 100 microseconds. Sends NoteOff only for active notes (typically 0-4). No resource exhaustion risk. |
| T-03-18 | Elevation of Privilege | SidebarPanel.getContentPanel() | accept | Returns internal JPanel reference. Only called by ReBeatBoxWindow (same codebase). Single-user desktop app — no privilege boundaries exist. |
| T-03-19 | Information Disclosure | PianoRollPanel live event subscription | accept | LiveNoteEventListener callback only calls repaint(). No note data is persisted or transmitted. No sensitive information in note events. |
| T-03-20 | Denial of Service | App.java startup flow | mitigate | Synthesizer initialization checked with System.exit(1) on failure. RealtimeReceiver and PlaybackController constructors throw on MIDI unavailable — caught with JOptionPane error dialog + System.exit(1). Receiver always available before wireEngine() is called. |
